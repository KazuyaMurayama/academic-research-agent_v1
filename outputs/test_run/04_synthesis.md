# エビデンス統合: 大規模言語モデルのハルシネーション低減手法

**実施日**: 2026-02-28
**分析対象**: Tier 1論文 10本 + Tier 2参照 15本

---

## 1. Tier 1論文 個別要約

### T1-1. A Survey on Hallucination in LLMs (Huang et al., 2024)
- **目的**: LLMハルシネーションの原理・分類体系・課題を包括的にサーベイ
- **方法**: 系統的文献レビュー、ハルシネーションの分類体系を提案
- **主要結果**: ハルシネーションを「事実性ハルシネーション」と「忠実性ハルシネーション」に大別。低減手法をプロンプト工学・RAG・ファインチューニング・デコーディング戦略に分類
- **限界**: 急速な進展により最新手法のカバレッジに限界

### T1-2. Siren's Song: A Survey on Hallucination in LLMs (Zhang et al., 2023)
- **目的**: LLMハルシネーションの定義・原因・対策の体系化
- **方法**: 系統的文献レビュー
- **主要結果**: ハルシネーションの原因を「データ起因」「学習起因」「推論起因」に分類。検出手法と低減手法を整理
- **限界**: 2023年時点の知見に基づくため、RAG・CoTの進展をカバーしきれない

### T1-3. RAG for LLMs: A Survey (Gao et al., 2023)
- **目的**: RAGの手法・アーキテクチャ・評価を包括的にサーベイ
- **方法**: 系統的文献レビュー、Naive RAG→Advanced RAG→Modular RAGの進化を体系化
- **主要結果**: RAGがハルシネーション低減に有効であることを確認。Retriever・Generator・Augmentation戦略の3要素を整理
- **限界**: ハルシネーション低減の定量評価が不十分

### T1-4. Benchmarking LLMs in RAG (Chen et al., 2023)
- **目的**: RAG環境でのLLMのハルシネーション傾向をベンチマーク
- **方法**: 4つのRAGタスク（ノイズ耐性・否定排除・情報統合・反事実耐性）で評価
- **主要結果**: ChatGPTが全体最良。ノイズ文書の存在がハルシネーションを増加させる。LLMは検索結果に過度に依存する傾向あり
- **限界**: 閉鎖型モデル中心の評価

### T1-5. SelfCheckGPT (Manakul et al., 2023)
- **目的**: 外部知識不要のハルシネーション検出手法を提案
- **方法**: 同一プロンプトに対する複数サンプリングの一貫性を評価（BERTScore、QA、n-gram）
- **主要結果**: サンプリングの一貫性が低い箇所がハルシネーションである可能性が高い。GPT-3のWikiBioタスクで高い検出精度
- **限界**: 検出のみで自動修正機能なし。サンプリングコストが高い

### T1-6. Chain-of-Verification (CoVe) (Dhuliawala et al., 2023)
- **目的**: LLMの自己検証によるハルシネーション低減
- **方法**: 4段階パイプライン: (1)ベースライン応答生成→(2)検証質問生成→(3)独立回答→(4)最終応答修正
- **主要結果**: リスト型質問でのハルシネーション率を大幅低減。Wikidata・MultiSpanQAで有効性を実証
- **限界**: 計算コストが高い。検証質問の質にパフォーマンスが依存

### T1-7. Verify-and-Edit (Zhao et al., 2023)
- **目的**: CoTの事実精度を外部知識で検証・編集
- **方法**: CoT推論の各ステップを外部知識ベースで検証し、不正確な部分を編集
- **主要結果**: 知識集約型QAタスクで精度向上。特にマルチホップ推論で効果的
- **限界**: 外部知識ベースの品質に依存

### T1-8. WikiChat (Semnani et al., 2023)
- **目的**: Wikipediaベースのグラウンディングでチャットボットのハルシネーションを停止
- **方法**: 7段階パイプライン: 主張抽出→ファクトチェック→検索→回答生成
- **主要結果**: GPT-4ベースのWikiChatを7B LLaMAに蒸留。事実正確性97.3%達成（GPT-4単体: 84.7%）
- **限界**: Wikipedia知識に限定。リアルタイム情報に弱い

### T1-9. RARR (Gao et al., 2023)
- **目的**: LLM出力の事後リサーチ・修正
- **方法**: (1)LLM出力から検証すべき主張を抽出→(2)検索エンジンでエビデンス取得→(3)エビデンスに基づき修正
- **主要結果**: 事実精度を向上しつつ、元の文体・構造を保持。属性付き修正が可能
- **限界**: 検索エンジンの品質に依存。計算コストが高い

### T1-10. Contrastive Learning Reduces Hallucination (Sun et al., 2023)
- **目的**: 対照学習による会話ハルシネーション低減
- **方法**: 正例（事実に基づく応答）と負例（ハルシネーション応答）の対照学習
- **主要結果**: 対話システムのハルシネーション率を有意に低減。知識グラウンド型対話で効果的
- **限界**: 高品質な負例データの生成が必要

## 2. クロス分析: 一致点と矛盾点

### 一致点（コンセンサス）

1. **RAGの有効性**: 全サーベイ論文（T1-1, T1-2, T1-3）がRAGをハルシネーション低減の主要アプローチとして特定。実験研究（T1-4, T1-8, T1-9）も有効性を実証
2. **多段階検証の効果**: CoVe（T1-6）、Verify-and-Edit（T1-7）、RARR（T1-9）いずれも「生成→検証→修正」の多段階パイプラインが有効
3. **ハルシネーションの分類**: 事実性（factuality）と忠実性（faithfulness）の2軸分類が確立（T1-1, T1-2）
4. **外部知識の重要性**: RAG（T1-3, T1-4）、WikiChat（T1-8）、Verify-and-Edit（T1-7）いずれも外部知識の参照がハルシネーション低減に不可欠

### 矛盾点・議論

1. **RAGの限界**: T1-4はRAGが「ノイズ文書によりむしろハルシネーションを増加させる」場合があることを報告。T1-3のサーベイ見解と部分的に矛盾
2. **自己検証 vs 外部検証**: SelfCheckGPT（T1-5）は内部一貫性のみで検出可能と主張。一方CoVe（T1-6）やVerify-and-Edit（T1-7）は外部知識が必要と主張
3. **コストと実用性のトレードオフ**: WikiChat（T1-8）の7段階パイプラインは高精度だが計算コスト大。実用性を重視するとシンプルなRAG（T1-3）で十分な場合も

## 4. テーマ別マッピング

### A. 検索拡張生成（RAG）ベースの手法
- **基本RAG**: 外部知識ベースから関連文書を検索し、プロンプトに付加（T1-3, T2-6）
- **Advanced RAG**: チャンク最適化、リランキング、クエリ変換（T1-3, T2-13）
- **グラフRAG**: 知識グラフを活用した構造化検索（T2-5, T2-10）
- **課題**: ノイズ文書の混入によるハルシネーション増加（T1-4）

### B. 自己検証・修正ベースの手法
- **Chain-of-Verification**: 自己生成した検証質問で回答を検証（T1-6）
- **Verify-and-Edit**: 外部知識でCoTステップを検証・編集（T1-7）
- **RARR**: 検索エンジンでエビデンスを取得し事後修正（T1-9）
- **特徴**: 精度は高いが計算コスト大

### C. 検出ベースの手法
- **SelfCheckGPT**: サンプリング一貫性による内部検出（T1-5）
- **HaluEval**: 大規模ベンチマークによる検出評価（T2-1）
- **TruthfulQA**: 事実性の標準評価ベンチマーク（T2-8）

### D. 学習段階での手法
- **対照学習**: 正例/負例ペアによるファインチューニング（T1-10）
- **知識蒸留**: 高精度モデル→小型モデルへの蒸留（T1-8 WikiChat）

## 5. エビデンスギャップ

1. **マルチモーダルハルシネーション低減**: 画像・音声を含むマルチモーダルLLMでのハルシネーション低減手法は未成熟
2. **リアルタイム知識更新**: 静的知識ベースに依存するRAGの限界。動的知識更新メカニズムの研究不足
3. **低コスト手法**: 多段階パイプライン（CoVe, WikiChat）の計算コスト削減
4. **ドメイン特化型低減**: 医療・法律等の高リスクドメインに特化した低減手法の体系的評価が不足
5. **長文生成でのハルシネーション**: 長い文書生成における累積的ハルシネーションの低減手法
6. **多言語対応**: 英語以外での低減手法の有効性検証が不足

## 6. 主要発見トップ3

1. **RAG + 自己検証の組み合わせが最も有望**: 外部知識の参照（RAG）と自己検証メカニズム（CoVe）を組み合わせることで、ハルシネーションを大幅に低減できる。WikiChatは事実正確性97.3%を達成
2. **ハルシネーションの原因は多層的**: データ品質、学習プロセス、推論メカニズムの3層にわたる原因があり、単一手法では完全解決困難。多層防御が必要
3. **検出と低減のギャップ**: ハルシネーション検出技術は進展しているが、検出結果を自動修正に活用するパイプラインはまだ発展途上。特にリアルタイム修正の実現が課題
