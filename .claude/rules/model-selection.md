# モデル使い分けルール

## 基本方針

| 用途 | モデル | 理由 |
|---|---|---|
| 計画立案・アーキテクチャ設計・複雑な論理推論 | **Opus** | 高度な推論が必要 |
| 実行・サブタスク・論文収集・レポート生成・その他大部分 | **Sonnet** | コスト効率と速度 |
| 軽量な読み取り・検索・確認タスク | **Haiku**（オプション） | 最速・最安 |

## 具体的な使い分け

### Opusを使う場面（`/research` の Phase 1: 検索戦略立案等）
- 複数の仮説を比較して最適解を選ぶ
- 100点満点スコアリングの採点基準設計
- エビデンス統合の総合判断
- 新しいルールや構造の設計

### Sonnetを使う場面（大半の処理）
- 論文収集・スクリーニング（Phase 2〜3）
- レポート生成（Phase 4〜5）
- ファイルの読み書き・編集
- git操作
- tasks.md / FILE_INDEX.md の更新

## Agentサブタスクでの指定方法

```python
# Opusを使うサブエージェント
Agent(subagent_type="Plan", model="opus", prompt="...")

# Sonnetを使うサブエージェント（デフォルト）
Agent(subagent_type="Explore", prompt="...")
```
